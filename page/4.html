When you deleted /lib on Linux while still connected via SSH (2022)

作者: todsacerdoti | 评分: 110 | 评论数: 17 | 发布时间: 2025-03-22 07:24
文章摘要

本文讨论了在Linux系统中误删`/lib`目录后导致的问题及恢复方法。由于`/lib`包含运行时所需的动态链接库，删除后会导致大多数命令无法执行。在没有静态`busybox`的情况下，可以利用当前shell的内置命令（如`bash`的`printf`）通过网络传输一个小型静态二进制文件（如`netcat`或自制的微型ELF程序）来恢复系统。自制的小型ELF程序可以通过纯汇编或极简C代码编译，并使用特殊的编译选项来最小化文件大小，以便通过`printf`命令构造并传输到受损系统中，从而恢复系统功能。

评论摘要
主要讨论点：Linux系统中因误操作（如使用rm -rf）导致数据丢失的风险及其应对方法 不同观点： • gleenn分享了一个因误用rm -rf导致几乎删除整个home目录的经历，幸运的是，由于一个较大的文件夹减缓了删除进程，最终仅丢失了部分数据。他以此为例，建议保留大文件作为缓冲以保护重要数据。 • eitland描述了一次因技术人员误执行"rm -rf /bin"导致系统问题的事件，最终通过从另一台相同服务器复制文件恢复系统。强调了编写防止此类错误的文档的重要性。 • inejge指出此类问题反复发生的原因是UNIX系统中root权限的绝对性及文件系统的无保护性，并提到未来可能通过不可变系统和受限执行环境来解决。 • ryao解释了技术术语"unlinked"文件的概念，指出在文件被进程使用时仍可通过特定工具恢复，提供了技术上的恢复方法。 • ofalkaed分享了两个因脚本错误导致误删重要目录的经历，强调了使用备份和注意脚本细节的重要性。 • lloeki描述了一次因误执行"umount /"导致系统混乱的事件，强调了早期系统对root权限缺乏保护的问题。 • zavec尝试通过创建可执行文件恢复系统，但未成功，不过认为这是有趣的经历。 • nurple展示了一种通过使用Nix包管理器来避免系统崩溃的方法，提供了另类的系统管理思路。 • throwanem建议在发现错误后暂停操作，给予自己一段时间思考，而不是立即继续操作。 • Fizzadar分享了一次通过复制文件系统恢复OpenVZ主机的事件，显示了手动修复的可行性。 • Dwedit认为在删除/lib等关键目录后，通过USB启动并重装包是更好的恢复方法。 • smw描述了一次在Solaris系统上通过硬链接恢复误移动/lib目录的事件，展示了静态链接二进制文件的优势。 • nullorempty分享了一次因误操作移除所有文件的执行权限的经历。 • jmclnx质疑了Linux是否曾包含一个静态链接的bash以应对此类问题，并提到可能与/bin和/usr/bin合并有关。 补充讨论： • 争议焦点之一是如何防止和应对因误操作导致的系统崩溃，包括技术手段（如恢复工具）和管理手段（如编写详细的操作文档）。 • 不同用户分享了各自的经验和恢复方法，展示了多种可能的解决方案和预防措施。 • 强调了在系统管理中保持谨慎操作和备份的重要性，以及未来可能的技术改进方向（如不可变系统）。

IronRDP: a Rust implementation of Microsoft's RDP protocol

作者: mikece | 评分: 508 | 评论数: 33 | 发布时间: 2025-03-21 15:35
文章摘要

IronRDP 是一个用 Rust 实现的微软远程桌面协议（RDP），注重安全性。它包括多个 Rust crates，支持多种视频编解码器如未压缩位图、RLE、RDP 6.0 压缩和 RemoteFX。IronRDP 提供一个基于这些 crates 的全功能 RDP 客户端，支持异步 I/O，并展示了一个简单的同步 RDP 客户端示例，可截图并保存为 BMP 文件。此外，项目还提供了在服务器上启用 RemoteFX 的指南。IronRDP 使用 Apache-2.0 和 MIT 双重许可，拥有 1.2k 个星标和 65 个分支，由 29 名贡献者开发。

文章介绍了新形式规范语言FizzBee，并通过建模互斥问题（即锁定问题）来测试该语言。作者Lorin Hochstein在FizzBee中模拟了两个进程的执行，并设置了一个不变量以确保任何时刻最多只有一个进程能进入关键区域。初始模型显示该不变量被违反，表明互斥未被实现。随后，作者加入锁机制以解决互斥问题，但遇到了死锁问题。FizzBee的模型检查器检测到死锁，并假设线程可能在任何语句后崩溃。为解决此问题，作者提出需要构建具备容错能力的锁定方案，例如引入租约机制以应对进程失败的情况。FizzBee基于Starlark（Python的子集），语法简洁，但缺少标签语法，这既是优点也是缺点。

Use Long Options in Scripts

作者: OptionOfT | 评分: 229 | 评论数: 21 | 发布时间: 2025-03-21 19:57
文章摘要

文章建议在脚本中使用命令行工具的长选项格式（如`--force`），而非短选项（如`-f`）。虽然短选项适合交互式命令行使用，但在脚本中，长选项更加直观易懂，有助于提高代码可读性。例如，在终端中可以输入`git switch -c my-new-branch`，但在脚本中应使用`git switch --create release-{today} origin/main`，以清晰表达操作意图。文章还提供了在脚本中使用长选项的示例代码。

评论摘要
