Peer-to-peer file transfers in the browser

作者: keepamovin | 评分: 31 | 评论数: 8 | 发布时间: 2025-03-12 12:08
文章摘要

FilePizza 是一个基于浏览器的对等（P2P）文件传输工具，利用 WebRTC 技术实现文件直接在用户之间传输，无需中间服务器，确保传输快速、私密且安全。其主要特点包括新版支持暗黑模式和现代浏览器技术，移动端兼容性强，上传者可监控和中止传输，支持多文件打包传输和流式下载。安全方面，提供密码保护和报告功能，所有传输自动加密。项目使用 Next.js、Tailwind、TypeScript 和 React 等技术开发，支持 Docker 部署。FilePizza 遵循 BSD 3-Clause 许可证发布。

评论摘要
主要讨论点：WebRTC本地 peer discovery 的可行性及相关工具的现状与问题 不同观点： • modeless 认为当前的 WebRTC 实现依赖于互联网连接和共享标识符（如特殊 URL），无法实现离线本地共享，希望有本地 peer discovery 的方法。 • mary-ext 担心这些工具（如 ShareDrop 和 Snapdrop）会像以前的一些项目一样被商业公司收购，提到这些工具最近被加密货币和AI公司LimeWire接管。 • Rygian 提到 Magic Wormhole 作为一个非浏览器内实现的解决方案，提供了一种不同的文件共享方式，并给出了项目的 GitHub 链接。 • arthursw 推荐 PairDrop，作为一个基于 WebRTC 的文件传输工具，并提供了项目链接。 • amelius 表示困惑于为何几十年前就应该解决的非商业性安全文件共享问题至今仍未实现，并询问IPFS项目的现状。 • janandonly 对基于WebRTC的文件传输尝试表示赞赏，但指出实际使用中经常遇到问题，同时提到正在测试一个有前景的工具keet，并提供了链接。 • enriquto 以怀旧的语气表示过去通过直接使用IP地址和netcat进行文件共享，暗示现代工具可能过于复杂。 补充讨论： • 多个评论提到了具体的工具和项目，如 Magic Wormhole、PairDrop、keet 和 IPFS，表明社区对实际解决方案的高度关注。 • 讨论中涉及到工具的收购和维护问题，显示出对工具长期可持续性的担忧。 • 评论中也反映了对离线文件共享功能的需求以及对现有工具性能的不满。 • 不同用户对现代技术与传统技术的对比，反映出对技术复杂性和使用便捷性的不同看法。

作者: ksec | 评分: 96 | 评论数: 10 | 发布时间: 2025-03-09 11:53
文章摘要

文章主要介绍了一个名为Open UI的项目，其目标是维护用户界面（UI）的开放标准并推动其应用。文章回顾了网页浏览器自1993年以来提供的形式控件和UI控件的发展历史，指出HTML5虽然增强了交互功能，但仍无法满足现代复杂网页项目的需求。当前开发者常依赖繁重的JavaScript框架创建自定义界面，导致可访问性差、性能低和安全问题。Open UI社区组致力于通过结合HTML、CSS、JS和Web API，为开发者提供现代自定义UI的技术基础，并提出新的标准建议以满足现代网页设计需求。
