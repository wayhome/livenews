Retro Boy: simple Game Boy emulator written in Rust, can be played on the web

作者: ingve | 评分: 131 | 评论数: 7 | 发布时间: 2025-03-20 21:54
文章摘要

Retro Boy 是一个用 Rust 编写的 Game Boy 模拟器，能够通过 WebAssembly 在网页上运行。它使用 wasm-pack 将 Rust 代码转换为 WebAssembly，并通过 Web Audio API 和 HTML Canvas 实现音频和图形渲染。该模拟器支持多种内存控制器（MBC1、MBC3、MBC5、HuC1）、实时时钟（RTC）以及电池支持的卡带存储。其网页前端使用 React 和 TypeScript 构建，支持全屏模式、暂停/恢复、作弊码管理等功能。要编译为 WebAssembly，需要安装 wasm-pack 并运行相应的脚本。该项目通过广泛的测试套件确保准确性，包括 Blargg 和 Mooneye 的测试 ROM 集合。

评论摘要
主要讨论点：Gameboy模拟器的性能与应用 不同观点： • [积极评价] [zeta0134] 对该Gameboy模拟器表示赞赏，尤其是使用Rust+WASM技术使其能在浏览器中运行。他提到音频模拟存在轻微问题（如不应出现的点击声），但整体实现非常出色。他还分享了自己用类似技术开发的NES模拟器。 • [浏览器兼容性问题] [catapart] 提到该模拟器在Firefox上运行缓慢且有音频问题，而在Chrome上表现良好。 • [其他平台表现] [GaggiX] 指出该模拟器在Ladybird浏览器上也能运行，但速度较慢。 • [简化开发与分享] [zeta0134] 强调使用WASM技术简化了开发和分享过程，可以直接通过链接分享游戏和接收反馈，无需复杂工具。 补充讨论： • [游戏推荐] [noeatnosleep] 作为一个没有从小接触游戏的玩家，请求推荐经典游戏进行尝试。 • [GB Studio比较] [turnsout] 提到GB Studio也可以导出网页播放器，并质疑该实现是否具有某些优势。 • [寻求资源] [GaggiX] 询问是否有好的Gameboy自制ROM资源。 争议焦点： • 模拟器在不同浏览器上的性能差异，尤其是在Firefox和Chrome之间的表现不一致，成为用户关注的焦点。

The Front End Treadmill

文章摘要
