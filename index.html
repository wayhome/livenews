JavaScript Temporal is coming

作者: SigmundurM | 评分: 1037 | 评论数: 61 | 发布时间: 2025-01-30 11:28
文章摘要

JavaScript Temporal即将推出，它将简化和现代化日期与时间的处理方式，特别有利于涉及调度、国际化或时间敏感数据的应用。与使用了近30年的老旧Date对象不同，Temporal解决了缺乏时区支持、解析不稳定和可变性等问题。Temporal提供了时间点、持续时间、时区和日历的表示，并支持200多个实用方法。核心概念包括Duration（持续时间）、Temporal.Instant（时间点）、Temporal.ZonedDateTime（带时区时间）和多种日期表示如Temporal.PlainDateTime（日期时间）、Temporal.PlainDate（仅日期）等。Temporal还简化了不同日历系统（如中文、希伯来、伊斯兰）的处理，并支持Unix时间戳操作。尽管跨浏览器稳定支持尚需时日，但MDN已提供了详尽的文档。

评论摘要
主要讨论点：JavaScript Temporal API 的优缺点及其使用中的具体问题 不同观点： • **支持并赞赏 Temporal API**： - **phiresky**：认为 Temporal 解决了旧 Date() API 的许多问题，尤其是时区处理上的复杂性。虽然 API 看起来更复杂，但这种复杂性是必要的，因为时间操作本身就很复杂。 - **qwertox**：赞赏 Temporal 处理时区转换和持续时间，尤其是 `withTimeZone()` 方法，解决了时区转换的常见需求。 - **stevage**：认为 Temporal 是一个巨大的改进，通过 polyfill 已经可以提前使用，没必要等待所有浏览器原生支持。 • **对 Temporal API 的一些功能缺失或命名不满**： - **qwertox**：指出 Temporal.Duration 缺少自定义格式化功能，如 `duration.format('H\hmm\m')`。 - **juancroldan**：对 "Temporal" 的命名不满，认为容易让人误解为与垃圾回收相关。 - **nikeee**：不满意 Temporal 使用引用相等性来进行比较，这导致无法将 Temporal 对象用作 Map 的键或放入 Set。 • **对 Temporal API 的未来和兼容性问题的关注**： - **ggregoire**：担心 Temporal 中时区数据的更新机制，尤其是浏览器未更新时区数据的情况下，用户可能会看到不正确的时间。 - **andyjohnson0**：对 JavaScript 长期使用有缺陷的 Date API 表示疑惑，并询问背后的原因。 - **nobleach**：希望 Temporal 和其他新特性（如 Records 和 Tuples）能够更快推出，并认为一些语法争论可以跳过。 补充讨论： • **与其他语言的时间处理库比较**： - **zackmorris**：提到 PHP 中的 Carbon 类，并建议使用 CarbonImmutable 来避免可变对象的问题。 • **对时间和时区的复杂性提出额外建议**： - **TheAceOfHearts**：建议添加检测用户更改时区的事件，因为程序员常错误地假设用户时区是固定的。 • **对 Temporal API 的实现细节和技术赞赏**： - **mstade**：赞赏 Temporal 提供的比较函数设计，认为这是新类型应该具备的良好设计。 • **对未来 Web 技术发展的期望**： - **culi**：幽默地表示期待嵌套 CSS 和 Temporal API 的全面实现，并希望专注于互操作性和 PWA API。 争议焦点： - **Temporal API 的复杂性与必要性**：部分开发者认为 Temporal 的复杂性是处理时间问题的必要之恶，而另一部分开发者则对 API 的某些功能（如格式化和比较）提出了改进期望。 - **Temporal 对象的引用比较问题**：一些开发者认为无法将 Temporal 对象用作 Map 键或放入 Set 是一个重大限制，而另一些开发者可能认为这是合理的设计选择。

作者: LeoPanthera | 评分: 176 | 评论数: 24 | 发布时间: 2025-01-30 19:37
文章摘要

**Stats** 是一个 macOS 系统监控应用，显示在菜单栏中，用于监控 CPU、GPU、内存、磁盘、网络、电池、传感器信息、风扇控制（已弃用）、蓝牙设备和多时区时钟等。支持 macOS 10.15 及以上版本，可以通过手动下载 DMG 文件或使用 Homebrew 安装。对于 M1 Mac，传感器数据需手动启用 HID 服务。Intel 处理器可显示 CPU 频率，需安装 Intel Power Gadget。若要降低 Stats 的能耗，可禁用一些模块如传感器和蓝牙。该应用支持多种语言，采用 MIT 许可协议。
