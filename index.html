Austral: A Systems Language with Linear Types and Capabilities (2022)

作者: yamrzou | 评分: 19 | 评论数: 0 | 发布时间: 2025-03-20 04:37
文章摘要

Austral是一种新的系统编程语言，可视为Rust的精简现代化版本或Ada的更新版。它具有强大的静态类型系统、线性类型、基于能力的安全性以及强模块化特性。本文介绍了Austral的设计目标和核心特性，强调了**简约性**和**严格性**。简约性指描述系统所需的信息量，Austral力求语言规范简短、编译器小、运行时薄，使整个语言易于理解和掌握。严格性则通过机械过程（如类型系统和形式验证）减少人为错误，避免其他语言中常见的设计缺陷。文章详细解释了线性类型及其安全性，以及基于能力的安保系统，展示了该语言如何实现安全和正确性。Austral的目标是让程序员能够完全掌握语言特性，避免复杂的语言谜题和设计缺陷。

The Pain That Is GitHub Actions

作者: qianli_cs | 评分: 99 | 评论数: 28 | 发布时间: 2025-03-20 03:37
文章摘要

文章作者Gerd Zellweger分享了使用GitHub Actions进行CI（持续集成）配置的痛苦经历。这是他们第三次重做CI设置，之前从GitHub Actions切换到Earthly（后因其停止维护而放弃），现在又回到GitHub Actions。他们的CI流程复杂，包括合并队列、多种运行器、Rust构建、Docker镜像和大量集成测试。作者指出了GitHub Actions的几个问题：在合并队列中强制状态检查的奇怪方式、隐藏的陷阱和不一致的行为，以及调试困难。 作者希望实现的一些良好实践（如自动修复小错误、确保测试和发布的工件一致、快速完成CI）在GitHub Actions中虽然可以实现，但设置过程非常繁琐。此外，GitHub的权限管理和安全模型也让作者感到困惑，特别是GITHUB_TOKEN的权限设置和依赖第三方Action时的安全隐患。文章最后提到，提升权限需要在工作流文件中手动设置，这在安全性上显得不太合理。

Bolt3D: Generating 3D Scenes in Seconds

作者: jasondavies | 评分: 145 | 评论数: 3 | 发布时间: 2025-03-19 22:30
文章摘要

本文介绍了一种名为Bolt3D的方法，能够在单个GPU上用6.25秒生成3D场景。该方法通过输入一张或多张图片，使用多视角扩散模型生成场景的外观和几何结构，然后利用高斯头（Gaussian Head）回归出Splatter图像。通过结合多个Splatter图像的3D高斯分布，构建出完整的3D场景。Bolt3D能够处理可变数量的输入图像，并在不使用反投影或修复机制的情况下生成未观察到的场景区域。与其他方法相比，Bolt3D具有生成能力，并显著降低了推理成本。实验结果显示，该方法在生成高质量3D场景方面优于其他基于优化的方法。
