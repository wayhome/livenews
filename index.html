Alibaba/T-HEAD's Xuantie C910

作者: mfiguiere | 评分: 125 | 评论数: 6 | 发布时间: 2025-02-04 05:40
文章摘要

Alibaba旗下全资子公司T-HEAD推出了一系列基于RISC-V架构的处理器核心，旨在降低对国外芯片技术的依赖，并为物联网和边缘计算等领域提供高性价比的芯片解决方案。其中，Xuantie C910是一款高性能RISC-V核心，支持向量扩展（RVV 0.7.1），可应用于人工智能、边缘服务器、工业控制和高级驾驶辅助系统等领域。C910采用四核集群设计，共享L2缓存，基于台积电12nm工艺，主频可达2至2.5 GHz，功耗较低。T-HEAD后续还发布了升级版C920核心，将RVV支持提升至1.0版本。C910的架构为3-wide、12级流水线、支持乱序执行，前端设计包括64 KB的指令缓存和预取数据，以提高指令 fetch 和分支预测效率。虽然C910的源代码已开放，但分析硬件细节对软件工程师而言仍具挑战性。

Httptap: View HTTP/HTTPS requests made by any Linux program

作者: alexflint | 评分: 510 | 评论数: 29 | 发布时间: 2025-02-03 16:28
文章摘要

**httptap** 是一个用于查看 Linux 程序发出的 HTTP/HTTPS 请求的工具。通过运行 `httptap -- `，可以捕获和显示任何 Linux 程序发出的 HTTP 请求和响应。例如，使用 `httptap -- curl https://monasticacademy.org` 可以显示 curl 请求的详细信息和返回状态。该工具无需 root 权限，也不需要修改系统配置或设置守护进程，不会影响系统中的其他进程。它利用 Linux 的网络命名空间等特定系统调用，因此目前仅支持 Linux，且难以移植到其他操作系统。 主要功能： - 捕获 HTTP/HTTPS 请求和响应。 - 无需 root 权限，不影响其他进程。 - 支持预构建二进制安装和 Go 安装。 - 示例展示了 curl 和 gcloud、kubectl 等命令行工具的 HTTP 请求捕获。 安装命令： ```bash curl -L https://github.com/monasticacademy/httptap/releases/download/v0.0.7/httptap_linux_x86_64.tar.gz | tar xzf - ``` 或使用 Go 安装： ```bash go install github.com/monasticacademy/httptap@latest ```

ArXiv LaTeX Cleaner: Clean the LaTeX code of your paper to submit to ArXiv

文章摘要

**arxiv-latex-cleaner** 是一个帮助研究者清理 LaTeX 代码以提交到 arXiv 的工具。它可以从一个包含所有代码的文件夹（如 `/path/to/latex/`）生成一个新的、适合打包上传到 arXiv 的文件夹（如 `/path/to/latex_arXiv/`）。主要功能包括： 1. **隐私保护**：删除辅助文件、注释及特定环境（如 `\begin{comment}`）。 2. **空间优化**：删除未使用的 `.tex` 文件和图片，并可调整图片大小以减小文件体积。 3. **TikZ 图片处理**：将 TikZ 图片替换为外部编译的 PDF 文件。 4. **自定义正则表达式替换**：支持通过 `cleaner_config.yaml` 文件定义复杂的模式替换规则。 安装方式：通过 `pip install arxiv-latex-cleaner` 或从源码安装。该工具要求 Python 版本 >= 3.9。

评论摘要
主要讨论点：在arXiv上发布论文时是否应该删除LaTeX代码中的注释和特定内容（如TikZ源代码） 不同观点： • [albertzeyer] 认为删除注释很遗憾，因为旧论文中的注释、讨论、思考过程以及被注释掉的部分（如不同的标题选项）提供了有价值的信息，能够让人更好地理解论文的写作和研究进展。他举例说明了一些论文中的有趣标题讨论，并指出像"Attention is all you need"这样的论文中也有很多有用的注释内容。 • [enriquto] 表达了对删除代码注释的不解，并表示自己喜欢查看.tex文件中的注释，认为这些注释有时可以带来乐趣，甚至希望有人会阅读自己留下的注释。 • [generationP] 质疑隐藏TikZ源代码的意义，认为这不仅增加了源文件的体积，还削弱了论文的可访问性，不利于其他研究者学习。 • [jpfr] 指出许多研究者通过参考他们喜欢的论文中的LaTeX习惯来学习LaTeX，包括TikZ图的代码。他希望工具仅用于删除无意披露的注释部分和减少文件大小，而不是全面修改LaTeX源代码。 补充讨论： • [firsching] 提供了一个工具链接（https://github.com/mo271/arxiv-comments），可以快速检查论文中留下的注释，暗示即使删除注释，也可以通过工具查看。 争议焦点：是否应该删除论文LaTeX源代码中的注释和TikZ源代码。支持保留注释者认为这些内容有学习和参考价值，而删除注释的支持者可能考虑的是隐私或文件整洁性。
